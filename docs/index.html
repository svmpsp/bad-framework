Welcome to BAD: Benchmarking for Anomaly Detection!¶

Warning

The BAD framework lets users execute arbitrary code on the server machine.

DO NOT deploy on production critical systems or open networks. Security features might be added in future releases.

BAD is a benchmarking framework for unsupervised anomaly detection algorithms.

BAD’s features include:

  • a simple command line interface.

  • a collection of benchmark data sets from the literature.

  • a convenient interface for hyperparameter tuning.

  • out-of-the-box support for distributed processing.

  • extensibility via custom python modules implementing the Candidate API.

BAD is built with lots of ❤ and Python.
